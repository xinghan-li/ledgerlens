## 一、Gemini 全面切换 & 结构化输出 | Gemini Migration & Structured Output

### 1. OpenAI 完全移除

- 删除所有 OpenAI 相关调用与引用，LLM 层统一为 Gemini-only。
- `prompt_manager.py` 中残留的 OpenAI model 引用改为 `settings.gemini_model`。

### 2. 结构化输出（Structured Output）

- 所有 Gemini 调用（`parse_receipt_with_gemini()`、`parse_receipt_with_gemini_vision()`、`parse_receipt_with_gemini_vision_escalation()`）统一添加 `response_schema=RECEIPT_OUTPUT_SCHEMA`。
- 在 API 层面强制 JSON schema，消除 LLM 输出解析失败的问题。

### 3. 上下文缓存（Context Caching）

- 新增 `_get_or_create_vision_cache()`，通过 `client.caches.create()` 缓存 system instruction + vision 规则，TTL 1 小时。
- 批量上传场景下，每次调用仅发送图片本身，系统指令从缓存读取。
- **预期效果**：重复 system prompt 的 token 消耗降低 50%+。

---

## 二、Prompt 外部化 | Prompt Externalization

- 核心 prompt 从代码内联改为 git 版控的模板文件：
  - `backend/app/prompts/templates/system_message.txt` — 系统指令
  - `backend/app/prompts/templates/prompt_template.txt` — 用户提示词模板
- `prompt_manager.py` 重构：通过 `_load_template()` 优先读取本地文件，店铺特定规则仍存储在 Supabase（可运维时热更新无需部署）。
- **设计原则**：核心 prompt 随代码走 → code review 可覆盖；运营规则存 DB → 无需发版。

---

## 三、地址 Grounding | Address Grounding with Google Search

- 新增 `backend/app/processors/enrichment/address_grounding.py`。
- **流程**：当 canonical DB 匹配失败时，使用 Gemini + Google Search grounding 查验/补全商户地址。
- **置信度逻辑**：
  - high/medium → 填充缺失字段；high 可覆盖已有 `address_line1`
  - low → 仅记录到 `_metadata.address_grounding`，不修改 receipt
- **已知问题**：Gemini API 不支持 `tools`（Google Search）与 `response_mime_type: "application/json"` 同时使用，导致 400 INVALID_ARGUMENT。已在 3/23 修复（移除 `response_mime_type`，改为 prompt 内要求 JSON）。

## 四、评估框架 | Evaluation Framework

- 新增 `backend/app/evaluation/` 模块：
  - `evaluators.py` — 9 个评估器：`evaluate_total`、`evaluate_subtotal`、`evaluate_tax`、`evaluate_item_count`、`evaluate_item_totals`、`evaluate_date`、`evaluate_merchant`、`evaluate_address_completeness`、`evaluate_sum_check`
  - `run.py` — CLI runner，加载 ground truth 与预测结果，输出 per-metric 平均分/完美率，per-receipt 失败详情，可选 JSON report
- **使用方式**：`python -m app.evaluation.run -g evaluation/ground_truth/ -p output/20260219/`
- **意义**：prompt 修改不再靠手动抽查，评估驱动迭代。

## 五、图片压缩 | Image Compression

- `workflow_processor_vision.py` 新增 `_compress_image_if_needed()` 辅助函数。
- 阈值 > 5 MB 时自动压缩：长边 2000px、JPEG quality 85（通常压至 ~2 MB）。
- **背景**：iPhone 照片常 5 MB+，Gemini Vision 按 token 计费，压缩后更快更省。
