## 一、Vision-First 识别流水线 | Vision-First Pipeline (Route B)

### 1.1 核心文件

- **`backend/app/core/workflow_processor_vision.py`**（新建，1099 行）：完整的 Vision-First Route B 流水线：
  1. 重复/限速/用户检查（与 legacy 一致）
  2. 创建 `receipt_status` 行，`pipeline_version='vision_b'`
  3. **PRIMARY**：Gemini Vision 直接读图 → 结构化 JSON
  4. 后端 sum check + item count check（独立于模型自报告）
  5. PASS → 存储并分类；模型标记 `needs_review`（仅 item count 问题）→ `needs_review` 不升级
  6. FAIL（sum check 不过或不明确）→ **ESCALATION**：并行 Gemini + OpenAI → 共识
     - 两者一致 → success
     - 两者不一致 → `needs_review`（预填共同字段，高亮冲突）
  7. Shadow legacy 始终在后台并行运行，用于 A/B 对比
- **`backend/app/main.py`**：新增 `POST /api/receipt/workflow-vision` 端点

### 1.2 Prompt 系统

- `VISION_PRIMARY_PROMPT`（主识别）：完整书记员提示，CENTS 整数输出规范，含重量计价、组合包装、折扣、税费等所有规则
- `VISION_ESCALATION_PROMPT`：升级共识提示，输出结构与 PRIMARY 一致
- `VISION_REVIEW_CHECKLIST`：AI 自检 prompt，用于模型自报告 `needs_review_flags`

### 1.3 相关后端改动

- **`workflow_processor.py`**：重构共用工具（`TimelineRecorder`、`_fail_output_payload`、`_save_output` 等）供 vision 模块复用
- **`sum_checker.py`**：扩展 sum check 逻辑以兼容 vision 输出格式
- **`gemini_client.py`**：新增 `parse_receipt_with_gemini_vision_escalation`
- **`llm_client.py`**：新增 `parse_receipt_with_openai_vision`
- **`config.py`**：新增 `GEMINI_ESCALATION_MODEL`、`OPENAI_ESCALATION_MODEL` 可选配置

## 五、未分类商品页面 | Unclassified Items Page

### 5.1 前端
