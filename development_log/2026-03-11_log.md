## 二、分类器 Bug 修复 | Categorizer Bug Fix

### `UnboundLocalError: has_summary` — receipt_categorizer.py:1252

- **根因**：`has_summary` 与 `has_items` 仅在 `if not force:` 块内赋值（第 1046 行），但第 1252 行在 `try` 块中无条件引用 `has_summary`。当调用方传入 `force=True` 时，两个变量未经赋值即被读取，Python 抛出 `UnboundLocalError`。
  - 首次发现：GCP Error Reporting（3/11, 9:02 PM），1 次出现，栈帧 `categorize_receipt /app/app/services/categorization/receipt_categorizer.py:1252`。

- **修复**：在 `if not force:` 块前增加默认值初始化：
  ```python
  has_summary = False
  has_items = False
  if not force:
      ...
  ```
  保证 `force=True` 路径下两个变量有确定值，逻辑不受影响（`force=True` 时会先清除旧数据再重写，`has_summary=False` 使第 1252 行直接走 `save_receipt_summary` 路径，与预期一致）。
